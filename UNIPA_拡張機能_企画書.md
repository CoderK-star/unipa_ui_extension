## 1. 背景と目的

UNIVERSAL PASSPORT（通称：ユニパ）は全国多数の大学に導入されている学生ポータルシステムであり、時間割・課題提出・掲示確認・成績照会など学生生活の根幹を担う。しかしアプリストアのレビューや口コミでは「操作性皆無」「仕様が複雑すぎる」「害悪」などの厳しい評価が並んでおり、ユーザビリティの課題は深刻な状況にある。

本企画では、サーバー側の変更を伴わない **ブラウザ拡張機能** というアプローチで、DOM操作・CSS注入・ローカルストレージを活用しながら、学生が直面する具体的な Pain Point を解消する。

## 2. ユーザー課題の整理（調査結果）

App Store・Google Play レビュー、知恵袋、Zenn 等の一次情報から頻出課題を抽出・重要度別に分類した。

### 課題一覧

| 優先度 | 課題 | 概要 |
|--------|------|------|
| 🔴 最重要 | セッション短すぎ問題 | 約30分で自動ログアウト。レポート入力中にセッション切れ→入力内容が全消滅するケースが多発 |
| 🔴 最重要 | ナビゲーション欠如 | ブラウザの「戻る」が使えないページ多数。1つ前に戻るだけでホームに飛ばされる |
| 🔴 最重要 | ファイルDL強制 | PDFや資料を閲覧するたびに必ずダウンロードが必要。インライン表示不可 |
| 🔴 最重要 | 掲示板が整理できない | 提出期限・掲示期間での並べ替えや絞り込みが不可。重要なお知らせを探しにくい |
| 🟡 重要 | 情報の分散・複雑UI | メニュー項目が過多。掲示・課題・シラバスが別々の場所にあり、全体像が掴みにくい |
| 🟡 重要 | 締切の視認性が低い | 課題の締切日時が目立たず、締切直前になってから気づくケースが頻発 |

## 3. ブラウザ拡張機能でできること／できないこと

### 実装可能な機能

| 機能カテゴリ | 備考 |
|-------------|------|
| DOM操作・CSS注入 | content script でページ要素の追加・変更・非表示が自由に行える |
| セッション自動延長（擬似） | 定期的に軽量リクエストを送り続けることでタイムアウトを遅延させられる |
| ローカルストレージへの一時保存 | テキスト入力を定期的にローカルへ保存し、ログアウト後も復元可能 |
| PDFのインライン表示 | ダウンロードリンクをブラウザ内プレビューURLに書き換え |
| 掲示板のソート・フィルタUI追加 | 既存テーブルのデータを取得してJS側で再レンダリング |
| 締切カウントダウン表示 | 期限日時を解析して残り時間バッジをDOMに注入 |
| デスクトップ通知 | chrome.notifications API で締切前プッシュ通知が送れる |
| キーボードショートカット | よく使うページへのホットキー設定 |

### 実装不可能な機能

| 機能カテゴリ | 理由 |
|-------------|------|
| サーバー側のセッション時間変更 | バックエンドの設定変更は不可。擬似的な延長のみ対応 |
| メンテナンス時間の変更 | サーバー側のスケジュールは変更不可 |
| API追加・バックエンド改変 | JASTのサーバーに直接触れる手段はない |

## 4. 機能設計

### 機能 1：オートセーブ＆セッション維持（最優先）

テキストエリア・入力フォームの内容を30秒ごとにローカルストレージへ自動保存。ページに再訪問した際に復元UIを表示。合わせてバックグラウンドで定期的に keepalive リクエストを送りセッション切れを防ぐ。

- 技術要素：`content_script` / `localStorage` / `keepalive fetch`

### 機能 2：パンくずナビゲーター（最優先）

現在のURL・ページ遷移履歴をトラッキングし、ページ上部にパンくずリストを注入。「戻る」ボタンが存在しないページにも汎用バックボタンを自動追加する。

- 技術要素：DOM注入 / `history API`

### 機能 3：掲示板強化（ソート・フィルタ・バッジ）（最優先）

掲示一覧ページを解析し、「締切が近い順」「未読のみ」「授業別」でのフィルタUIをページ上部に注入。既読管理をローカルで持ち、未読バッジを表示する。

- 技術要素：DOM再レンダリング / `localStorage`

### 機能 4：PDFインラインビューア（中優先）

資料ダウンロードリンクにマウスオーバーするとプレビューポップアップを表示。クリック時はブラウザ内PDFビューアで開くようリンクを書き換え、ダウンロードを省略できる。

- 技術要素：URL書き換え / `iframe` preview

### 機能 5：締切ダッシュボード（フローティングウィジェット）（中優先）

すべての課題・提出期限をスクレイプして、画面右下にフローティングウィジェットとして表示。残り時間に応じて色が変化（緑→黄→赤）し、24時間前にはデスクトップ通知を送る。

- 技術要素：DOM注入 / `chrome.notifications` / `chrome.alarms`

### 機能 6：クイックナビゲーション（コマンドパレット）（中優先）

Ctrl+K（Mac: ⌘+K）でコマンドパレットを起動。「時間割」「掲示板」「課題提出」などのキーワード検索で目的のページに即座に移動。深いメニュー階層を一発でスキップ。

- 技術要素：`KeyboardEvent` / DOM注入

### 機能 7：UIテーマカスタマイズ（低優先）

ダークモード切替・フォントサイズ調整・カラーテーマ選択をポップアップから設定可能。視覚的な疲労を軽減し、長時間使用でも快適な環境を提供する。

- 技術要素：CSS変数注入 / `chrome.storage`

## 6. 技術スタック

| 領域 | 採用技術 | 理由 |
|------|----------|------|
| 拡張機能フレームワーク | Manifest V3（Chrome Extensions API） | 現行のChrome標準規格。Edgeにも対応 |
| フロントエンド | Vanilla JS + Shadow DOM | UNIPAのスタイルと競合させないためShadow DOMでカプセル化 |
| 状態管理・永続化 | chrome.storage.local + localStorage | オフライン時もデータを保持可能 |
| 通知 | chrome.notifications + chrome.alarms | バックグラウンドでの締切通知を実現 |
| スタイル注入 | CSS Custom Properties | テーマ変数を一元管理し動的切替を容易に |

## 7. リスクと対応策

| リスク | 対応策 |
|--------|--------|
| UNIPAのDOM構造が大学ごとに異なる | 主要大学のセレクタを調査してフォールバック処理を実装。設定画面でユーザーがセレクタをカスタマイズできるモードも提供 |
| UNIPAのアップデートによるDOM変更 | セレクタの変更に即応できるよう設定ファイルを外部化。バージョン検出機能で自動警告 |
| keepaliveが大学サーバーに不審と判断される | 最小限のリクエスト（5分間隔・軽量エンドポイント）に留め、オプトインとして実装 |
| 個人情報の取り扱い | すべてのデータはローカルのみで処理。外部サーバーへの送信は一切行わない |
